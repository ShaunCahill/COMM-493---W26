INPUT FORMAT: [Overall Quality, Living Area sqft, Year Built, Basement sqft, Garage sqft]

HOW TO TEST:
1. Open the Lambda function in the AWS Console
2. Click "Test" tab
3. Paste ONE test event at a time (just the JSON block, not the label text)
4. Click "Test" and check the response

WHAT TO LOOK FOR:
- Tests 1 and 3 should return DIFFERENT closest_cluster values (they
  represent opposite ends of the market). Test 2 could match either.
- With K=2, you will see clusters 0 and 1. With K=3, you will see 0, 1, and 2.
- The Lambda uses quality-conditional defaults: a quality-9 home gets
  realistic premium medians for ALL non-user features (Exter Qual,
  Kitchen Qual, Year Built, etc.), not just global medians
- If all tests return the SAME cluster, check CloudWatch logs for:
    * The quality level being used for defaults
    * The PCA values (PC1, PC2) - they should differ significantly
    * Any errors in the preprocessing steps
- If PCA values are nearly identical across all tests, the issue is
  likely in QUALITY_MEDIANS (did you populate it correctly?)

  may have been trained with more than 2 PCA components (check the
  tuning output in your notebook for num_pca_components)
